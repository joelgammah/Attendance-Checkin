Frequently Asked Questions

Common questions and troubleshooting for the Attendance Check-In system.

General Questions

Q: How do I reset my password?

A: Password reset is handled by the authentication provider (Auth0). Click "Forgot Password" on the login screen and follow the email instructions.

Q: Can I check in to an event early?

A: Yes, but only within the check-in window. By default, check-in opens 15 minutes before the event start time. Contact the organizer if you need to check in earlier.

Q: What happens if I lose my check-in QR code?

A: Contact the event organizer. They can provide you with the check-in URL or token directly. You can also access the event through your dashboard if you're logged in.

Q: Can I check in multiple people at once?

A: No, each person must check in individually with their own account. This ensures accurate attendance tracking and security.

Organizer Questions

Q: How do I add attendees to my event?

A: Create the event first, then share the QR code or check-in URL with intended attendees. They will appear in your attendee list after checking in.

Q: Can I modify an event after creating it?

A: No, events cannot be modified once created. If you need to change an event, you'll have to create a new one and ask admin to delete the old one.

Q: How do I export attendance data?

A: Go to your event page and click "Export CSV" or access the API endpoint: GET /api/v1/events/{id}/attendance.csv

Q: Can I customize the check-in window?

A: No, for now the default is 15 minutes.

Technical Issues

Q: The QR code won't scan. What should I do?

A: Try these steps:

  1. Ensure good lighting and steady hands
  2. Clean your camera lens
  3. Try manually checking in using the event token
Q: I'm getting "CORS error" in the browser console

A: This is usually a backend configuration issue. Ensure:

  • The backend CORS_ORIGIN includes your frontend URL
  • Both frontend and backend are running on expected ports
  • No ad blockers or security extensions are interfering
Q: "Callback URL mismatch" error when logging in

A: The Auth0 application settings need to include your callback URL. Add these to your Auth0 app:

  • Allowed Callback URLs: http://localhost:5173/callback (dev) or your production URL
  • Allowed Logout URLs: Your frontend base URL
  • Allowed Web Origins: Your frontend base URL
Q: Database connection errors

A: Check these items:

  • Verify DATABASE_URL environment variable is correct
  • Ensure database server is running and accessible
  • Check network connectivity and firewall rules
  • Verify SSL settings (add ?sslmode=require for production)

Development & Deployment

Q: How do I run the app locally?

A: Follow the setup in README.md:

  1. Clone the repository
  2. Set up Python virtual environment and install backend dependencies
  3. Install frontend dependencies with pnpm
  4. Start backend: uvicorn app.main:app --reload
  5. Start frontend: pnpm dev
Q: How do I deploy to production?

A: See RENDER.md for detailed deployment instructions using Docker and Render. Key steps:

  1. Build Docker images for frontend and backend
  2. Push images to Docker Hub
  3. Deploy to Render with proper environment variables
  4. Configure Auth0 with production callback URLs
Q: How do I update the documentation?

A: Edit files in frontend/public/docs/, then rebuild and redeploy the frontend Docker image. The docs are bundled with the frontend and served statically.

Security & Privacy

Q: Is my data secure?

A: Yes. The system uses:

  • HTTPS encryption for all data in transit
  • JWT tokens for secure authentication
  • Role-based access control
  • Secure password hashing
  • Database connection encryption
Q: Who can see my attendance data?

A: Attendance data is only visible to:

  • Event organizers (for their events only)
  • System administrators
  • The attendee themselves (their own records)
