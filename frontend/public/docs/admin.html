Admin Guide

Administrative functions for managing users, organizations, and system-wide settings in the Attendance Check-In system.

Admin Dashboard

Admin users can:

  • View and manage users and events
  • Create, edit, and delete user accounts
  • Assign user roles (organizer, attendee)
  • View system-wide attendance data
  • Access audit logs

User Management

Creating Users

Admins can create new users through the admin panel. Required fields:

  • Email: Must be unique, serves as the username
  • Full Name: Display name for the user
  • Role: organizer or attendee

Role Permissions

  • Admin: Full system access, user management, all organizations
  • Organizer: Create/manage events, view attendees, export data
  • Attendee: Check into events, view assigned events
⚠️ Important: Admin role grants full system access. Only assign to trusted users who need system-wide management capabilities.

Monitoring & Logs

Health Checks

  • GET /healthz - Basic application health
  • GET /api/v1/health - Detailed system status

Audit Logs

The system tracks important actions:

  • User login/logout
  • Event creation/modification
  • Check-in activities
  • Admin actions (user creation, role changes)

Troubleshooting

Common Admin Issues

  • Cannot access admin panel: Verify user has admin role assigned
  • Users cannot log in: Check Auth0 configuration and callback URLs
  • Database errors: Verify DATABASE_URL and connection
  • CORS issues: Ensure CORS_ORIGIN matches frontend URL exactly

Support Contacts

For technical issues or system administration help:

  • Check system logs in the hosting environment
  • Review the deployment documentation in RENDER.md
  • Contact the development team via project repository
